# 🎰 ランチバンディット

きょうのランチ、探索か活用か。多腕バンディット(割引UCB)が毎日のランチを提案するWebアプリ。

**📱 使う: https://yasut0ra.github.io/lunch-bandit/** — スマホならブラウザの「ホーム画面に追加」でアプリとして入ります(PWA、オフライン起動対応)。データは端末のlocalStorageに保存されます。

「いつもの店」と「気になる新規店」のトレードオフを、UCBスコアの内訳(期待値+探索ボーナス)ごと可視化するのがコンセプト。行った店に⭐を付けるほど賢くなり、しばらく行っていない店には探索ボーナスが戻ってくる。

## 動かし方

```bash
npm install
npm run dev    # http://localhost:5173
npm test       # アルゴリズムのユニットテスト
npm run build  # 型チェック + 本番ビルド
```

データはすべて localStorage(`lunch-bandit:v1`)。せってい画面からJSONエクスポート/インポート可能。

## アルゴリズム

各店のスコアは割引UCB:

```
score_i = μ̂_i + c · √( ln N / n_i )
```

- 評価⭐1〜5を `(r-1)/4` で [0,1] に正規化
- **日数減衰**: 訪問の重みは `γ^(経過日数)`。γは「味の記憶」設定(忘れない=1 / ゆっくり=0.999 / ふつう=0.995 / 移り気=0.99)
- `n_i` = 減衰込み有効訪問回数 + 事前擬似回数(n₀=1)、`N` = 候補全体の合計
- **未訪問店**は楽観的事前分布(μ₀=0.7)のみを持ち、大きなボーナスで自然に上位へ
- `c` は「探索ぐあい」設定(控えめ=0.2 / ふつう=0.4 / 冒険=0.7)
- 完全同点(未開拓店同士)だけ、日付シードの決定的ハッシュで日替わりローテーション。ジッターはスコアに足さないので表示ptと順位は常に一致する

実装は [src/lib/bandit.ts](src/lib/bandit.ts)(純関数、テストは [src/lib/bandit.test.ts](src/lib/bandit.test.ts))。

## 構成

```
src/
  lib/
    bandit.ts     # 割引UCBスコアリング + 理由文生成(コア)
    types.ts      # Restaurant / Visit / Settings + ジャンル定義
    storage.ts    # localStorage永続化・エクスポート/インポート
    sample.ts     # サンプルデータ(10店・26訪問)
    format.ts     # 日付・星の表示ヘルパー
  components/
    RecommendView.tsx  # きょう:統計タイル・開拓日和バナー・スコアバー付きランキング
    HistoryView.tsx    # きろく:日付グループの訪問履歴
    StoresView.tsx     # おみせ:登録・休止・削除
    SettingsView.tsx   # せってい:c/γプリセット・データ管理・数式解説
    RecordSheet.tsx    # 記録用ボトムシート(⭐入力)
```
